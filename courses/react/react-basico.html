Curso básico de react js

React cumple su función como biblioteca ya que para utilizar su código se debe importar. También es un Framework aunque las convenciones de cómo debe ser organizado todo no son estrictas.
React es declarativo, lo que quiere decir que se le indica qué debe hacer pero no cómo debe hacerse, ahorrando de esta manera muchos pasos.
React está estructurado por componentes que son como pequeños bloques de lego que al ser unidos forman aplicaciones de React. Estos componentes pueden tener estilos, ser enlazados a eventos y sus estados pueden ser modificados. Con React también se tiene la ventaja de que será escrito una sola vez y podrá ser utilizado en aplicaciones web, móviles, entre otras.

Create React App

Create React App app es una aplicación moderna que se usa desde una línea de comando. Antes de ella se configuraba todo el entorno manualmente lo cual tomaba mucho tiempo. 


                //Instalar de forma global
                npm install -g react-create-app

                //Crear proyecto
                react-create-app nombre_proyecto

En este punto se estará instalando React y otras herramientas, también se configurará el entorno usando Webpack. Una vez se instala todo entra a la carpeta src donde estará todo el código fuente de la aplicación, siendo el más importante index.js que es el punto de entrada a la aplicación.
Finalmente, para correr la aplicación se usa el comando: 


                npm run start

Babel: Traduce Javascript moderno (JSX) a un Javascript que todos los navegadores puedan interpretar.
Eslint: Lee el código y avisa de errores.

Para desistalar Creat React App : npm uninstall -g react.

ReactDOM.render

  • React y ReactDOM trabajarán en conjunto.
  • React como análogo a createElement.
  • ReactDOM a appendChild.
  • ReactDOM.render() toma dos argumentos: Qué queremos renderizar y dónde lo queremos renderizar.
  • Siempre que escribas JSX es requisito importar React.

Ahora veremos como seria una aplicación que añade un elemento al DOM sin usar React y otra donde si implementemos React, (Estos cambios se haran en el index.js de la aplicación).

Sin usar React 


                //Se crea un elemento html, en este caso un h1
                const element = document.createElement("h1")
                  
                //A ese elemento se le agrega un texto                  
                element.innerText = "Hello, this is a title"              
                
                //El siguiente paso es donde se añadira ese elemento, 
                se añadira al contenedor que tenga el id "app", el unico elemento que hay en el index.html                  
                const container = document.getElementById("app")
                  
                //Se añade con el metodo appendChild                  
                container.appendChild(element)
Con React

                /*Lo primero es importar React, que esta en el módulo(carpeta que ya tenemos descargada al instalar react-create-app)
                este elemento se comportara como createElement de JS Vanilla.*/ 
                import React from "react"
                  
                /*Otro elemento a importar es ReactDom del módulo react-dom, que se comportara como appendChild.*/
                import ReactDOM from "react-dom"

                const element = <h1> This is a title with JSX and React </h1>
                const container = document.getElementById("app")

                //ReactDOM.render(que_queremos_renderizar, donde_se_renderizara)

                ReactDOM.render(element, container)

JSX

Es una extensión de JavaScript creada por Facebook para el uso con la biblioteca React. Sirve de preprocesador (como Sass o Stylus a CSS) y transforma el código generado con React a JavaScript.
JSX tiene su alternativa que es React.createElement pero es preferible JSX porque es mucho más legible y expresivo. Ambos tienen el mismo poder y la misma capacidad.

React.createElement recibe 3 argumentos:
  • El tipo de elemento que estamos creando.
  • sus atributos o props.
  • Y el children que es el contenido.
Ejemplo: 

                React.createElement("h1", {href:"https://carloszabala.com"},"ir al home")

Jsx es azúcar sintética (Syntactic Sugar! es como el Async Await de las promesas de JS).

Veamos en un ejemplo la diferencia entre JSX y createElement. 


                //JSX
                const jsx = (
                  <div>
                    <h1>My name is ...</h1>
                    <p>This is a paragraph</p>
                  </div>
                )

                //CreateElement
                const element = React.createElement("div", {}, 
                  React.createElement("h1", {}, "My name is...",
                  React.createElement("p", {}, "This is a paragraph"
                ))

Jsx es mas expresivo, mejor legible, más claro. Jsx antes de llegar al navegador, se debe interpretar con webpack y babel, para poder transformar las líneas de código "fácil" que escribimos a js para que lo puede entender el navegador.

¿Qué es un componente?

Los componentes en React son bloques de construcción.
Las aplicaciones hechas con React son como figuras de Lego. Junta varias piezas (componentes) y puedes construir un website tan pequeño o grande como quieras. Los componentes serán barras de búsquedas, enlaces, encabezados, el header, etc.

Componente vs elemento
Un elemento es a un objeto como un componente es a una clase. Si el elemento fuera una casa, el componente serían los planos para hacer esa casa.

Identificación de componentes
Para identificarlos debes hacerte las siguientes preguntas:

  • ¿Qué elementos se repiten? Estos son los elementos en una lista o los que comparten aspecto visual y su funcionalidad
  • ¿Qué elementos cumplen una función muy específica? Estos sirven para encapsular la lógica y permiten juntar muchos comportamientos y aspectos visuales en un solo lugar.

Unidad básica de desarrollo --> elemento (átomo).
Lo que forma un website --> componente.

Creación de un componente

  • Es una buena práctica que los componentes vivan en su propio archivo y para ello se les crea una carpeta (src/components).
  • Todos los componentes requieren por lo menos el método render que define cuál será el resultado que aparecerá en pantalla.
  • El source de las imágenes en React puede contener direcciones en la web o se le puede hacer una referencia directa importándola. Si se importa deben usarse llaves para que sea evaluado.
A continuación, se creara el componente badge: 

                //Lo primero siempre importar React 
                import React from "react"

                "Component", para poder usar funcionalidades de los componentes, como el método render.
                class Badge extends React.Component{

                  //Todos los componentes requieren al menos 1 método que es obligatorio y es el render,
                  lo que hace es que define cual será el resultado que se verá en pantalla
                  render(){
                    return(

                //Por último hay que exportar ese componente

Props

Los props que es la forma corta de properties son argumentos de una función y en este caso serán los atributos de nuestro componente como class, src, etc. Estos props salen de una variable de la clase que se llama this.props y los valores son asignados directamente en el ReactDOM.render().

Los props son como atributos en HTML, primero hay que pasarle los props al componente de la siguiente forma: 

                < Badge nombreProps = valorProps />

                < Badge 
                  firstName = "Carlos"
                  lastName = "Zabala"
                  jobTitle = "Frontend Engineer"
                  twitter = "czabala847"
                />
Y en el componente se usan por medio de this.props. 

                //Puedes usar destructuring para acceder fácilmente a las props. El nombre de la variable debe ser igual al nombre del prop.
                render(){
                  const {firstName, lastName, jobTitle, twitter} = this.props
                }

Páginas en React

Las páginas en React son componentes y conseguir distinguirlas nos servirá para saber que es un componente que adentro lleva otros componentes.
Al escribir los props no importa el orden en el que lo hagas, únicamente importa el nombre.

Para crear la página es igual al componente, la diferencia es que esta contendra más componente, los cuales deberan ser importados mediante import.
La página será la que se importe en el archivo principal index.js.

Eventos

  • React dispone de eventos. Cada vez que se recibe información en un input se obtiene un evento onChange y se maneja con un método de la clase this.handleChange.
  • Los elementos button también tienen un evento que es onClick.
  • Cuando hay un botón dentro de un formulario, este automáticamente será de tipo submit. Si no queremos que pase así hay dos maneras de evitarlo: especificando que su valor es de tipo button o manejándolo desde el formulario cuando ocurre el evento onSubmit.
El evento se usa como atributo(props) 

                //Usando el evento de onChange dentro de la caja de texto
                <input
                  onChange = {
                    this.handleChange
                  }
                  type = "text"
                  name = "firstName"
                />

                //Evento de click
                <button 
                onClick = {this.handleClick}
                className = "btn"
                >
                  click           
                </button> 

                //Evento de submit
                <form onSubmit = {this.handleSubmit}>        
                </form> 

                //Para truncar el evento de submit, en el metodo handleSubmit colocar:
                handleSubmit(e){
                  e.preventDefault()
                }

Si el evento es onChange el método debe ser handleChange.

Estados

Hasta esta clase todos los componentes han obtenido su información a través de props que vienen desde afuera (otros componentes) pero hay otra manera en la que los componentes pueden producir su propia información y guardarla para ser consumida o pasada a otros componentes a través de sus props. La clave está en que la información del state a otros componentes pasará en una sola dirección y podrá ser consumida pero no modificada.

  • Para guardar la información en el estado se usa una función de la clase component llamada setState a la cual se le debe pasar un objeto con la información que se quiere guardar.
  • Aunque no se ve, la información está siendo guardada en dos sitios. Cada input guarda su propio valor y al tiempo la está guardando en setState, lo cual no es ideal. Para solucionarlo hay que modificar los inputs de un estado de no controlados a controlados.

Guardar dentro del state, la información digitada en el formulario gracias al evento onSubmit y el método handleSubmit. 


                //Es una función flecha, para así no tener problemas con el this, si se hace con una función normal hay que usar bind, para especificarlo.
                handleChange = (e) => {
                  this.setState({
                    firstName : e.target.value  
                  })
                }
Guardar todos los campos del formulario: 

                /*[e.target.name] crea variables con el nombre de los inputs del formulario (firstName, lastName, email)
                e.target.value, guarda el valor ingresado en las cajas de texto dentro de las variables(atributos) nombrados anteriormente*/
                this.setState({
                  [e.target.name] = e.target.value
                })

El state en React es un objeto que deriva de la clase React Component. Puede definirse a sí mismo como un objeto que almacena las propiedades observables que controlan el comportamiento del componente. (como en este caso el value de los inputs de los formularios).

para declarar una variable como key del set state es necesario escribirla entre corchetes cuadrados: 


                [e.target.name]

Ejemplo de un input controlado:


                <input
                  onChange = {this.handleChange}
                  type = "text"
                  name = "firstName"
                  value = {this.state.firstName}
                />

Importante inicializar el state


                state = {
                  firstName : "",
                  lasttName : "",
                  jobTitle : "Designer",
                  email : "",
                  twitter : ""
                }

                //Iniciando el state dentro del constructor
                constructor(props){
                  //En React al especificar el constructor, se debe también llamar al constructor de la clase padre (Component).
                  super(props)

                  state = {
                    loading : true ,
                    data : undefined ,
                    error : null
                  }

                }

Resumen formularios controlados: darle el poder a react que controle todo, y que lo que se muestre en las cajas de texto sea state.value, y no lo que digite el usuario, claro que debe estar el evento onChange escuchando cada que digitamos o borramos una letra. Levantamiento del estado

Levantar el estado es una técnica de React que pone el estado en una localización donde se le pueda pasar como props a los componentes. Lo ideal es poner el estado en el lugar más cercano a todos los componentes que quieren compartir esa información. Algo interesante que le da el nombre a React es su parte de "reactivo" ya que cada vez que hay un cambio en el estado o en los props que recibe un componente se vuelve a renderizar todo el componente y todos sus descendientes.
Pasar el state a la página y no en el componente, para que así la información se pueda compartir en otros componentes 


                //En la página, que tendra el state, se crea el método handleChange
                handleChange= (e) => {
                  this.setState({
                    form : {
                      desestructurar objeto
                      ...this.state.form,
                      [e.target.name] : e.target.value
                    }
                  })
                }

                //El método handleChange, se le pasara como prop al componente hijo
                < BadgeForm
                  listenChange = {this.handleChange}
                  //Se pasa la información almacenada como prop.
                  dataForm = {this.state.form}
                />

En los inputs del componente hijo, al evento dispararse el evento onChange, se llamara al método handleChange del componente padre. 


                <input
                  //Se llama al método del componente padre pasado como prop.
                  onChange = {this.props.listenChange}
                  type = "text"
                  name = "firstName"
                  value = {this.props.dataForm}
                />

Al modificar directamente el state, sin utilizar la función setState, se pierde el poder de React como, por ejemplo: componentDidMount, no verá esto como una actualización porque se modifica directamente.

Una forma sencilla y clara de modificar el state sin alterar su mutabilidad es mediante el siguiente ejemplo:
En el state hay un array de géneros de películas, el array tiene un nombre, una descripción y un array de películas. Lo que se quiere es modificar el array de películas por los datos de una API y actualizar el state sin mutarlo. 


                //Primero clonar el array de géneros, hacerle una copia
                const genreNew = [...this.state.genre]

                //O con el método de los arrays slice().
                const genreNew = this.state.genre.slice()
                
                //Se recorre el array nuevo, para asignarle al array de películas la data que arrojo la api.

                genreNew.forEach(genreItem => {
                  genreItem.movies = dataAPI.movies
                })

                //Se actualiza el state, con el nuevo array.
                this.setState({
                  genre : genreNew
                })

React Router

Las aplicaciones que se trabajan en React son llamadas single page apps. Esto es posible gracias a React Router que es una librería Open Source.
Multi Page Apps: Cada página implica una petición al servidor. La respuesta usualmente tiene todo el contenido de la página.
Single Page Apps (SPA): Aplicaciones que cargan una sola página de HTML y cualquier actualización la hacen re-escribiendo el HTML que ya tenían.
React Router (v4): Nos da las herramientas para poder hacer SPA fácilmente. Usaremos 4 componentes:

  • BrowserRouter: es un componente que debe estar siempre lo más arriba de la aplicación. Todo lo que esté adentro funcionará como una SPA.
  • Route: Cuando hay un match con el path, se hace render del component. El component va a recibir tres props: match, history, location.
  • Switch: Dentro de Switch solamente van elementos de Route. Switch se asegura que solamente un Route se renderize.
  • Link: Toma el lugar del elemento <a>, evita que se recargue la página completamente y actualiza la URL.

Gracias a SPA No perdemos toda la información, no hay que pintar todo de nuevo, es mucho más rápido. 


                //Brouser Router hace que el switch, link, route funcionen este siempre debe estar declarado en lo más alto de la página.
                
                //Route representa una dirección de internet, Recibe 2 prop path (donde se va a renderizar)
                y component (que componente se va a renderizar, este componente debe ser una página)
                < Route path = "/" component = {Home} />
              
                //Switch igual al switch de js, solo se ejecuta una opción de muchas.

Dividiendo la aplicación en rutas

Lo primero es instalar React Router por npm. 


                npm install react-router-dom

Se comienza con browser Router, esto habilita que la app funcione con react Router, se crea un nuevo componente, es allí donde se usará react Router y donde se añadirán las rutas. Se crea un archivo en la carpeta de components llamado App.js.

Y dicho componente se importa en el index.js. 


                import App from "../components/App"

Dentro de App, se crea el componente, pero esta vez no sera una clase si no un componente funcional, ya que no necesitaremos del state. Se procede a import React Router, y el primer componente como ya se dijo debe ser BrowserRouter. 


                //App iria quedando así:
                
                import React from "react"
                
                //Importando React Router:
                import {BrowserRouter} from "react-router-dom"

                function App(){
                  return(

                    //Usando BrowserRouter:
                    <BrowserRouter>
                    
                    </BrowserRouter>
                  )
                }

                export default App

Dentro de esos tags se van a añadir las rutas, que serán las paginas de nuestra aplicación. 


                //Esas páginas hay que importarlas antes, (donde se importa todo)
                <Route path = "/badges" component = {Badge} />
                <Route path = "/badges/new" component = {BadgeNew} />

Si se dejara esto así, se generaria el siguiente error "Uncaught Invariat Violation: A <Router>, may have only one child element."
Esto se debe a que BrowserRouter, solo acepta un solo hijo, y le estamos pasando dos (2 Route). Se podría colocar un div y dentro de él colocar los Route. Aunque esto generaría 2 errores, uno que se recargaría toda la página algo que se quiere evitar y segundo las páginas se renderizarían a la misma vez. En vez de usar un div se utiliza Switch, este toma la dirección que está en el navegador y renderizara una sola página, la primera que coincida con dicha ruta, para asegurarnos de que la ruta que se va a renderizar sea la correcta, que sea exacta se coloca el prop exact. 

                
                import React from "react"
                import {BrowserRouter, Route, Switch, Link} from "react-router-dom"

                function App(){
                  return(
                    <BrowserRouter>
                      <Switch>
                        <Route exact path = "/badges" component = {Badge} />
                        <Route exact path = "/badges/new" component = {BadgeNew} />
                      </Switch>
                    </BrowserRouter>
                  )
                }

                export default App

Solo quedaria reemplazar las anclas por Link.


                <Link to = "/badge/new" className = "btn btn--primary" > 
                  New Badge
                </Link>

Si quieren pasar props a cada componente deberían borrar el atributo component y cambiarlo por render. render quedaría un arrow function para que no se ejecute al instante y espere la decisión del usuario. Dentro del segundo paréntesis damos por implícito el return. 


                <Route exact path = "/badges/new" render = {() => ( <Badges props = {this.props} )} />

Layout

  • React.Fragment es la herramienta que te ayudará a renderizar varios componentes y/o elementos sin necesidad de colocar un div o cualquier otro elemento de HTML para renderizar sus hijos. Al usar esta característica de React podremos renderizar un código más limpio y legible, ya que React.Fragment no se renderiza en el navegador o solo colando (< >).
  • El 404 es la ruta que se renderizará cuando ninguna otra coincida con la dirección ingresada. 

                //Si el usuario ingresa una ruta incorrecta podemos mostrar el 404 de la siguiente forma:
                <Route component = {NotFound} />

Al contruir páginas con react, debemos ver que componentes se estan reptiendo que código se repite, para poder evitarlo, hay páginas donde el navbar y el footer se repiten para todo el sitio. Este código repetido se puede llevar a un solo sitio para que se comparta en todas las paginas a eso se le llama layout o plantilla en común para todas las páginas.

El Layout sera un componente funcional que se importar en App.js donde esta el React Router. 


                //Todo lo que tiene Switch queda dentro del componente Layout
                <BrowserRouter>
                  <Layout>
                    <Switch>
                      <Route exact path = "/badges" component = {Badge} />
                      <Route exact path = "/badges/new" component = {BadgeNew} />
                    </Switch>
                  </Layout>
                </BrowserRouter>

Ahora para que esto funcione y se pueda renderizar las páginas dentro del componente Layout, se deben retornar sus hijo, que sera Switch. 


                function Layout(props){
                  return(
                    //con React.Fragment -> esto es un componente especial que react entiende que va a hacer que
                    podemos regresar varios elementos, pero en realidad va a hacer que regresemos uno.
                    Esto sería imaginario, ya que en el html no se muestra. 
                    <React.Fragment>
                      <NavBar />
                        {props.children}
                      <Footer />
                    </React.Fragment>

Ciclo de vida de un componente

Cuando React renderiza los componentes decimos que entran en escena, cuando su estado cambia o recibe unos props diferentes se actualizan y cuando cambiamos de página se dice que se desmontan.

  • Montaje
    • Representa el momento donde se inserta el código del componente en el DOM.
    • Se llaman tres métodos: constructor (donde se inicializan los estados o valores), render (momento preciso donde react calcula este elemento y lo introduce en el código), componentDidMount (una vez lo del render ocurra, react envía una señal, ejecuta el método componentDidMount, ya con esto el código es visible en pantalla).
  • Actualización
    • Ocurre cuando los props o el estado del componente cambian.
    • Se llaman dos métodos: render, componentDidUpdate (recibe dos argumentos, los props que tenía anteriormente y el estado que tenía ).
  • Desmontaje
    • Nos da la oportunidad de hacer limpieza de nuestro componente. Cuando el componente sale de escena, ejemplo cuando se va de una página a otra.
    • Se llama un método: componentWillUnmount (lugar perfecto para limpiar memoria).

Practica del ciclo de vida

Un componente pasa por muchos estados.

El constructor recibe unos props, esos props se le pasan al constructor de la clase padre (super). El constructor es el lugar donde se deben inicializar los datos.

                      }
                    ]
                  }
                }

componentDidMount, que será el que se ejecuta después del render. Es en este método el ideal si queremos hacer una petición aun API, ya que esa llamada se ejectura una vez nuestros componentes esten cargados en pantalla. 


                componentDidMount(){
                  console.log("this is a componentDidMount")
                }

Render, dentro de el estarán todos los componentes que se mostraran en pantalla. 


                render(){
                  return(

                      </h1>

                1. constructor()
                2. render()
                3. componentDidMount()

Las llamadas a una API siguen un patrón similar siempre que las hacemos, cada llamada consta de tres estados:

  • Error: se debe dejar un mensaje para el usuario para arreglar el error o volver a intentarlo.
  • Data: los datos nos pueden llegar de dos formas, o en error o con los datos requeridos.

Las funciones no tienen un estado propio que manejar como ciclos de vida a los que deben suscribirse, mientras tanto las clases sí cuentan con ello. React tiene un feature llamado Hooks que permite que las funciones también tengan features que solamente tienen las clases.

  • useState: Para manejo de estado. Regresa un arreglo de dos argumentos.
  • useEffect: Para suscribir el componente a su ciclo de vida.
  • useReducer: Ejecutar un efecto basado en una acción.
  • Custom Hooks:Usamos los hooks fundamentales para crear nuevos hooks custom. Estos hooks irán en su propia función y su nombre comenzará con la palabra use. Otra de sus características es que no pueden ser ejecutados condicionalmente (if). 

                const [state, setState] = React.useState(initialState)

El resultado de ese filtro se guardará en una variable, y filtramos el array de badges, usando la función de los array filter.
Cuya condición para filtrar los badges son los que contengan en el nombre lo que se esta escribiendo en la caja de texto. Para comparar que una palabra esta contenida en otra se usa la función includes(). 


                const filteredBadges = badges.filter(badge => {
                  return badge.firstNameincludes(query);

Se pueden ignorar las mayúsculas y minúsculas para evitar problemas de filtrado. 


                return badge.firstName.toLoweCase().includes(query.toLowerCase())

Este filtrado puede generar un cuello de botella cuando la lista sea mucho más grande ya que cada vez esta calculando y renderizando cada vez que se digite una letra en la caja de texto, para eso se puede usar otro hook uno que ya trae react. Se llama useMemo, el cual calcula, busca los resultados y los guarda por decirlo así, ya que con esto al hacer nuevamente la misma búsqueda el ya tiene guardado esos datos de la búsqueda (argumentos) y no tiene que volver a calcularlos, con esto los mostrara muy rápidamente ya que los conoce

Recibe como argumentos una función, la que ya teníamos, y segundo unos argumentos, que si cambian realizara el filtrado, el cambio y si no cambian mostrara los que ya tiene guardado. Los argumentos que siempre cambian en este caso son la lista de badges, y los datos que ingresan en la caja de texto.

Antes hay que crear un estado, cuyo valor inicial será la lista de badges completas Luego en el useMemo guardamos el result en el estado que creamos. 


                const [filteredBadges, setFilteredBadges] = React.useState(badgeList)

                  const result = badges.filter(badge => {
                    return badge.firstName.toLoweCase().includes(query.toLoweCase())
                  })
                  setFilteredBadges(result)
                }, [badge, query])
