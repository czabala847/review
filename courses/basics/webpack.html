Introducción a Webpack

Webpack es un empaquetador para Javascript y sus amigos. Convierte módulos con dependencias en archivos estáticos que los navegadores entienden.

Nos permite empaquetar, optimizar los diferentes módulos Javascript y sus dependencia en nuestro proyecto. Es usado en proyectos basados en Javascript como: React, Vue, Angular entre otros.

  • User Experience. Se logra con una aplicación que:
    • Funcione.
    • Sea rápida.
    • Cumpla sus necesidades.
    • Se actualice.
    • Responda a sus interacciones.
    • Producto de calidad.
  • Developer Experience
    • Escribir aplicaciones de manera eficiente.
    • Tener un código limpio.
    • Aplicar tecnología para resolver sus problemas.
    • Tener un conjunto de reglas y convenciones.
    • Entorno de desarrollo optimizado en productividad.

Webpack es un conjunto de reglas que se configuran para saber como esta funcionando nuestro stack y como manipular el stack.

Configurando un nuevo proyecto de Javascript

Para inciar un proyecto en JavaScript debemos tener instaldo node y npm en nuestro ordenador. Una dependencia son los módulos que vamos a necesitar para que nuestra aplicación funcione.

En nuestro proyecto existen dependencias del tipo desarrollo y del tipo core con sus respectivas versiones, por eso usaremos NPM para administrarlas e instalar Webpack que viene a ser una dependencia (de desarrollo) más en nuestro proyecto de Javascript. 

                  
                    npm install webpack --save-dev --save-exact
  • --save: para que la dependencia se instale en package.json
  • --save-d (-D): para que la dependencia se instale como dependencia de desarrollo
  • --save-exact (-E): para instalar la misma versión

Creando nuestro primer bundle con Webpack

Vamos a instalar otra dependencia llamada webpack-cli, la API que expone webpack en forma de CLI (Command Line Interface) que nos va a permitir interactuar y configurar Webpack desde la terminal. Sin esto no podemos manipular ni escribir webpack desde la consola. 

                  
                    npm install webpack-cli --save-dev --save-exact

Es muy importante que nuestras dependencia o módulos se instalen en nuestros proyectos y no en el entorno global, para así no tener problemas con las versiones si tenemos varios proyectos. Pasar nuestros archivos a webpack(archivos que se van a transpilar) 

                  
                    npx webpack --entry ./index.js --output ./bundle.js

El archivo salida (output) es el archivo que le daremos al cliente, en este caso es el archivo que ira en el src de la etiqueta script del HTML.

El comando webpack tiene una bandera llamada --mode que nos permite cambiar entre los modos producción y desarrollo. Recuerda que por defecto nos pone en modo producción si no la especificamos.

Iniciando un webpack.config

Al ir creciendo nuestra configuración de Webpack iremos agregando cada vez más banderas a nuestros comandos y terminará como una línea gigante en la terminal. ¿Cómo hacemos que esa línea sea muy pequeñita, personalizable y escalable? Por medio de un archivo llamado por defecto webpack.config.js.

Este archivo permite importar módulos usando el formato commonJS y recibe por lo menos dos configuraciones básicas, un entry y un output. 

                  
                    /* 
                      * En el archivo webpack.config.js 
                      * Path para usar rutas relativas en cualquier S.O
                    */

                    const path = require("path");

                    /* Exportar archivo */
                    module.exports = {
                      entry: './index.js',
                      mode: 'development',
                      output: {
                        /* __dirname: Directorio raíz donde corre el comando  */
                        path: path:resolve(__dirname),
                        filename: 'bundle.js'
                      }
                    }

Se debe configurar en el package json el script para ejecutar el programa, comúnmente con npm run build.

Múltiples puntos de entrada

Nuestra aplicación no solo dependerá de un solo archivo, sino de varios, de muchos puntos de entrada, múltiples manejadores de varias páginas. Para eso en nuestro archivo de configuración webpack.config.js en la parte de entry podemos pasar un objeto con varias rutas. 

                  
                    entry: {
                      home: path.resolve(__dirname, 'src', 'js', 'index.js' )
                      precios: path.resolve(__dirname, 'src', 'js', 'precios.js' )
                      contacto: path.resolve(__dirname, 'src', 'js', 'contacto.js' )
                    }
                    output: {
                      path: path.resolve(__dirname, 'dist')
  
                      /*
                        * Para generar nombres de archivos usamos el template [name]
                        * [name] corresponde a los nombres de key del objeto que se le pasa a entry
                      */
                      filename: 'js/[name].js'
                    }

Introducción al uso de plugins

Los Plugins sirven para extender las capacidades de webpack y dar más poder a los loaders. Los loaders interpretan los tipos de archivos que tenemos y ya!!!. Pero para poder manipular esos archivos, para hacer lo que queremos necesitamos de plugins.

Plugins CSS

Para manipular archivos .css usaremos dos plugins:

  • mini-css-extract-plugin: Este complemento extrae CSS en archivos separados. Crea un archivo CSS por archivo JS que contiene CSS. Admite carga bajo demanda de CSS y SourceMaps.
  • html-webpack-plugin: Complemento que simplifica la creación de archivos HTML para servir sus paquetes. 
                  
                    npm i -D -E mini-css-extract-plugin html-webpack-plugin

En el archivo webpack.config.js:

                  
                    const MiniCSSExtractPlugin = require('mini-css-extract-plugin');
                    const HTMLWebpackPlugin = require('html-webpack-plugin');

                    module.exports = {
                      ...

                      plugins: [
                        new htmlWebpackPlugin({
                          inject: true,
                          template: "./public/index.html",
                          filename: "./index.html",
                        }),
                        new MiniCssExtractPlugin({
                          filename: "assets/app.css",
                        }),
                      ]
                    }

Si tenemos varios archivos HTML:

                  
                    new HtmlWebpackPlugin({
                      template: "./src/templates/home.html",
                      filename: "index.html",
                      chunks: ['home'] 
                    }),
                    new HtmlWebpackPlugin({
                      template: "./src/templates/contact.html",
                      filename: "contact.html",
                      chunks: ['contact'] 
                    })

Loaders para CSS y preprocesadores de CSS

Los elementos que se utilizaran serán los siguientes:

  • mini-css-extract-plugin
  • css-loader
                  
                    npm i mini-css-extract-plugin css-loader -D

Lo que se debe hacer es que en el template principal el archivo index.html, quitar el enlace del archivo css, y en el archivo princial del proyecto index.js importar los estilos. 

                   
                     import "./css/styles.css"

Archivo de configuración de webpack

                   
                     const MiniCssExtractPlugin = require('mini-css-extract-plugin')

                     module: {
                       rules: [
                        {
                          test: /\.css$/i,
                          use: [MiniCssExtractPlugin.loader, 'css-loader']
                        }
                       ]
                     },
                     plugins: [
                      new MiniCssExtractPlugin(),
                     ]

Implementando sass en el proyecto.

                   
                     npm i node-sass sass-loader

Configurando en el archivo principal de webpack.

                   
                    module: {
                      rules: [
                       {
                         test: /\.css|.scss$/i,
                         use: [MiniCssExtractPlugin.loader, 'css-loader', 'sass-loader']
                       }
                      ]
                    },

Importar archivo index.scss en el archivo principal del proyecto index.js 

                   
                     import "./css/index.scss";

Webpack Dev Server

Webpack Dev Server se encarga de montar un servidor para un entorno de desarrollo que se actualiza automáticamente cuando guardemos cambios en el proyecto. 

                  
                    npm i webpack-dev-server -D

La configuración se debe hacer en el archivo de webpack para desarrollo webpack.config.dev.js 

                  
                    module.exports = {
                      ...
                      devServer : {
                        contentBase: path.join(__dirname, 'dist'),
                        compress: true,
                        historyApiFallback: true,
                        port: 3006,
                        open: true
                      }
                      ...
                    }

En el package hay que pasar la nueva configuración

                  
                    "scripts" : {
                      ...
                      "start" : "webpack serve --config webpack.config.dev.js"
                      ... 
                    }

Soporte de Javascript moderno

Javascript es un lenguaje moderno en evolución, siempre agregando nuevas funciones. El problema es que al ser interpretado en el navegador, no tenemos control sobre que versión de Javascript soportan y por lo tanto que funciones.

Para poder usar Javascript moderno y tener una buena Developer Experience sin afectar la User Experience, existe Babel. Babel transpila nuestro código moderno de Javascript a una una versión que todos los navegadores pueden entender.

Babel será un nuevo loader para intersertar los archivos de Javascript. 

                  
                    module.exports = {
                      ...
                      module: {
                        rules: [
                          test: /\.m?js$/,
                          use: 'babel-loader',
                          exclude: /node_modules/
                        ]
                      }
                      ...
                    }

exclude: Que directorios o tipos de archivos que, aunque sean Javascript no queremos que se le aplique babel-loader.

Babel requerirá de configuración, ésta puede ser declarada dentro del webpack, pero se recomienda hacerlo dentro de un archivo llamado .babelrc que leerá babel a la hora de ser configurado por webpack. 

                  
                    /* Archivo .babelrc */
                    {
                      "presets": [
                        "@babel/preset-env"
                      ],
                      "plugins": [
                        "@babel/plugin-transform-runtime"
                      ]
                    }

Instalar las dependencia:

                  
                    npm i -D -E @babel/core babel-loader @babel/preset-env

                    /* Nueva forma */
                    npm i babel-loader @babel/core @babel/preset-env @babel/plugin-transform-runtime -D

Soporte imágenes, fuentes y videos

Para las imagenes ya no es necesario la instalación de loader, ya que webpack nos ofrece la posibilidad de manejar archivos sin instalar ningún paquete por npm. 

                  
                    output:{
                      path: path.resolve(__dirname, 'dist'),
                      filename: 'main.js',
                      assetModuleFilename: 'assets/images/[hash][ext][query]'
                    },
                    module: {
                      rules:[
                        {
                          test:/\.png/,
                          type: "asset/resource"
                        }
                      ]
                    }

Y ya podemos importar imagenes en los archivos de JavaScript. 

                  
                    import imgTwitter from "../assets/images/twitter.png";

                    <img src="${imgTwitter}" />

Fuentes

Hay que descargar los recursos de las fuentes que se usaran en el proyecto, google font proporciona la opción de descargar las fuentes en formato woff ir. 

                  
                    /*css*/ 
                    @font-face{
                      font-family: 'Ubuntu';
                      src: url('../assets/fonts/ubuntu-regular.woff2') format('woff2')
                        url('../assets/fonts/ubuntu-regular.woff') format('woff');
                      font-weight: 400;
                      font-style: normal;
                    }

Tambien hay que mover los archivos de src a dist. Para eso se necesitan url-loader. 

                  
                    npm i url-loader file-loader -D

Configuración en webpack.

                  
                    module: {
                      rules:[
                        {
                          test:/\.(woff|woff2)$/,
                          use: {
                            loader: 'url-loader',
                            options: {
                              limit: 10000,
                              mimetype: "application/font-woff",
                              name: "[name].[ext]",
                              outputPath: "./assets/fonts/",
                              publicPath: "./assets/fonts/",
                              esModule: false
                            }
                          }
                        }
                      ]
                    }

Optimización: hashes, compresión y minificación de archivos

Una de las cosas más importantes por las que se usa webpack es por la optimización de archivos como css, js, imagenes entre otros recursos. Las siguientes dependencias nos ayudaran en ese proceso. 

                  
                    npm i css-minimizer-webpack-plugin terser-webpack-plugin -D

webpack nos comunica que actualmente terser-webpack-plugin viene incluido desde webpack 5 

                  
                    /*Archivo webpack*/
                    const CssMinimizerPlugin = require('css-minimizer-webpack-plugin');
                    const TerserPlugin = require('terser-webpack-plugin');

                    module.exports = {
                      optimization: {
                        minimize: true,
                        minimizer: [
                          new CssMinimizerPlugin(),
                          new TerserPlugin(),
                        ]
                      }
                    }

Hash

Los recursos que se guardan en memoria cache suceden cuando el navegador entra a un sitio por primera vez detecta los recursos y los guarda. Por ello la siguiente vez sera mucho más rápido porque estarán en memoria

  • La desventaja esta cuando sacamos una nueva versión, porque tendrán un mismo nombre evitando que se descargue los nuevos cambios, por lo tanto, el usuario no recibirá los nuevos cambios.
  • Para que no haya conflictos con la cache una vez que tengamos nuestro proyecto en producción es importante darles un hash para cada nueva versión 
                  
                    module.exports = {
                      output: {
                        filename: '[name].[contenthash].js'
                      }

                      /*Hash en las fuentes*/
                      module: {
                        rules: [
                          {
                            test: /\.(woff|woff2)$/,
                            name: '[name].[contenthash].[ext]'
                          }
                        ]
                      }

                      /*plugins*/
                      plugins: [
                        new MiniCssExtractPlugin({
                          filename: "assets/[name].[contenthash].css"
                        })
                      ]
                    }

Alias

Nos permiten otorgar nombres paths específicos evitando los paths largos. 

                  
                    /*Creación*/
                    module.exports = {
                      ...
                      resolve:{
                        alias: {
                          '@utils' : path.resolve(__dirname, "src/utilis/"),
                          '@templates' : path.resolve(__dirname, "src/templates/"),
                          '@styles' : path.resolve(__dirname, "src/styles/"),
                          '@images' : path.resolve(__dirname, "src/assets/images/"),
                        }
                      }
                      ...
                    }

                    /*Uso*/
                    import imgTwitter from '@images/twitter.png'

Variables de entorno

Es importante considerar las variables de entorno va a ser un espacio seguro donde podemos guardar datos sensibles. Por ejemplo, subir llaves al repositorio no es buena idea cuando tienes un proyecto open source, se debe instalar el paquete de npm dotenv-webpack. 

                  
                    npm i dotenv-webpack -D

Se debe crear los archivos .env, que es dondé viviran las variables de entorno, este archivo no se debe subir a ningún repositorio y debe ser confidencial y el archivo .env-example, para saber las variables que se usan en el proyecto. 

                  
                    /*Archivo env*/
                    API = https://randomuser.me/api/

Configuración en webpack

                  
                    const Dotenv = require("dotenv-webpack");

                    plugins: [
                      new Dotenv()
                    ]

Ejemplo de uso de las variables de entorno.

                  
                    const API = process.env.API;

Copia de archivos con Webpack

Hay veces que necesitamos mover archivos a la carpeta dist (distribución), para hacerlo con webpack necesitamos del plugin copy webpack plugin. 

                  
                    npm i copy-webpack-plugin
                  
                
                  
                    //webpack.config.js
                    const CopyWebpack = require("copy-webpack-plugin");

                    plugins: [
                      new CopyWebpack({
                        patterns: [
                          {
                            from: path.resolve(__dirname, "src/assets/images"),
                            to: "assets/images",
                          },
                        ],
                      }),
                    ],

Webpack Bundle Analyzer

Cuando tenemos un proyecto es buena idea poder revisar su impacto en tamaño por ese motivo webpack nos ofrece un paquete para poder verificar y analizar el tamaño del bundle final. 

                  
                    npm i webpack-bundle-analyzer -D

Esta configuración solo es para el modo de desarrollo.

                  
                    const BundleAnalyzerPlugin = require("webpack-bundle-analyzer");

                    plugins: [
                      new BundleAnalyzerPlugin(),
                    ]

Correr el recurso, y ver donde podemos hacer enfasis en mejorar nuestro código. 

                  
                    npx webpack --profile --json > stats.json

                    npx webpack-bundle-analyzer stats.json

Resolver problemas de codificación con Powershell en windows. Referencia, ejecutar el siguiente comando: 

                  
                    npx webpack --profile --json | Out-file 'stats.json' -Encoding OEM
