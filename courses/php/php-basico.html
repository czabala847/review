Curso básico de PHP

PHP es un lenguaje de programación de propósito general de alto nivel que fue diseñado especialmente para el desarrollo de aplicaciones web.
Es interpretado, multiplataforma, open source, el cual ha sido muy popular en los últimos años.

¿Qué NO es PHP?

  • No es un lenguaje compilado, por lo cual siempre tendrás que llevar juntos tu código y tu interprete.
  • No esta diseñado para realizar aplicaciones de escritorio.

Para trabajar con PHP instalaremos un entorno de desarrollo llamado XAMPP, no es un entorno pensado para producción.

Sintaxis de PHP

Siempre que usemos PHP usaremos lo siguiente: <?php ?> todo lo que pongamos dentro de esto será lo que el servidor va a interpretar como código php, lo que esté fuera lo ignorará.

Si queremos escribir código php en nuestra vista HTML tendremos que cambiarle la extensión al archivo por .php porque nuestro servidor esta configurado a solo interpretar archivos PHP. Solo las partes dentro de <?php ?> van a ser interpretadas y su código fuente no será visible desde el navegador.

Variables

Una variable puede ser una pequeña cajita en la que puedes almacenar un valor y este lo pueden usar para realizar alguna operación.

Para declararla usaremos el símbolo de $ y en seguida el nombre, este puede ser un _ o una letra.

PHP no es estáticamente tipado, es decir que no tenemos que decirle qué tipo de dato es esa variable. Además, es débilmente tipado porque podemos fácilmente cambiar el tipo de dato, es decir PHP ejecuta una conversión de datos interna. 


                  $var1 = 1;
                  $name = 'Carlos';
                  $lastName = 'Zabala';

                  #Con comillas dobles PHP trata de interpretas la variables
                  $fullName = "Nombre completo $name $lastName";

Al momento de trabajar con PHP una cosa muy importante es hacer debugging a nuestras variables, para ello utilizamos la función var_dump(); pasándole por parámetro la variable a revisar. 


                  var_dump($name);

Con el comando echo mostramos un mensaje en el navegador. 


                  <?php echo "Esto se mostrara en el documento, me llamó $name" ?>

Tipos de Datos en PHP

Escalares

  • boolean: Representa solamente un valor verdadero (true) o falso (false).
  • Integer: Representa un número entero positivo, negativo o 0.
  • float o double: Representa un número de punto flotante, existen problemas de precisión con los números flotantes debido a la naturaleza binaria de las computadoras.
  • string: Representa una cadena de caracteres. Existen 4 formas de representar una cadena. Las 2 principales son usando comillas simples o comillas dobles.

Los otros tipos de datos son los compuestos, como por ejemplo: array, objetos, etc.

Arreglos

Estas variables que almacenan más de un dato se conocen como arreglos y su sintaxis se va a indicar con [ ] (corchetes).

La estructura de arreglos en PHP es conocida como mapa, lo que quiere decir que tiene una composición de llave valor. Además, un arreglo puede contener más arreglos y cada uno de ellos seguirá la misma estructura. 


                $array = [
                  [
                    'llave' => 'Valor',
                    0 => 'Contenido'
                  ],
                  [
                    'otroArray' => 'Otro Valor'
                  ]
                ];

                #Usando el array
                $array[0][llave] #Resultado: Valor

Algo que debes saber es que en PHP podrás almacenar diferentes tipos de datos en un mismo arreglo.

Condicionales y Ciclos

Las condiciones nos permiten tomar decisiones en el código, si se cumple la condición entonces se ejecutarán ciertas instrucciones sino se cumple se ejecutarán otras. Estas se denotan por la instrucción if else.

Los ciclos funcionan de la mano con las condiciones, en este caso si se cumple la instrucción se estará ejecutando repetidas veces una instrucción dada.

for es una forma más simplificada de usar todos los elementos que componen los ciclos. 


                  #Ejemplo de recorrer un array con for
                  <?php

                    for($i = 0; $i < count($array); $i++ ){
                      #Template HTML
                      echo "
                          <li class = \"name-class\" >
                            <h5> {$array[$i]['title']} </h5>
                            <p> {$array[$i]['description']} </p>
                          </li>
                        "
                    }

                  ?>

Foreach

El constructor foreach proporciona un modo sencillo de iterar sobre arrays. foreach funciona sólo sobre arrays y objetos, y emitirá un error al intentar usarlo con una variable de un tipo diferente de datos o una variable no inicializada. más información. 


                  #En cada iteración, el valor del elemento actual
                  #se asigna a $element y el puntero interno del array avanza una posición
                  foreach ($arreglo as $element){}

                  #Asigna la clave del elemento actual a la variable $clave en cada iteración
                  foreach ($arreglo as $clave => $valor){}

Agregando archivos externos

Para organizar mejor nuestro código podemos separar archivos php.
Se usa la palabra reservada include para hacer que el archivo index incluya el archivo php externo, si lo encuentra lo incluye, pero si no nos mostrará un warning. Existe otro llamado require que si no lo encuentra nos muestra un error en todo el archivo.

Los métodos include y require ejecutan el código del archivo cada vez que lo incluyen, esto puede traer errores en la ejecución de tu código si tienes archivos con funciones pues te dirá que no puedes declarar dos veces una función con el mismo nombre. Para resolver esto existen include_once y require_once que obligan a incluir una sola vez el archivo. 


                include('otraArchivo.php');
                require('otraArchivo.php');
                require_once('otraArchivo.php');

Programación Orientada a Objetos

La programación orientada a objetos nos ayudará a estructurar mejor nuestros programas, lo que lo hace tener código más reutilizable y mantenible. Una clase es una plantilla o definición de algo. Y una instancia es la representación concreta de la clase.

Encapsulamiento será el nivel de visibilidad que queramos darle a alguna variable, para ello podemos utilizar los modificadores de acceso private, public y protected.

Con la palabra reservada this estaremos haciendo referencia a la variable que pertenece a la clase.

El método constructor nos permitirá inicializar valores por default, así como también pasar datos como parámetro al momento de inicializar un objeto. Todas las funciones que tienen __ antes del nombre de la función se conocen como métodos mágicos. 


                class NombreClase {
                  private $atributo1;
                  public $atributo2;
                  public $atributoN;

                  #Contructor de la clase
                  public function __construct($atributo1, $atributo2){
                    $this->setAtributo1($atributo1)
                    $this->atributo2 = $atributo2
                    $this->atributoN = true
                  } 
                  
                  public function setAtributo1($parametro){
                    if($parametro == '' ){
                      $this->atributo1 = 'N/A'
                    }
                    else{
                      $this->atributo1 = $parametro
                    }
                  }
                  
                  public function getAtributo1(){
                    return $this->atributo1
                  }
                }

                #Se crea la instancia de la clase
                $objeto1 = new NombreClase('valor1', 'valor2');
                $objeto1->setAtributo1('Valor');

Herencia

La herencia permite que ciertas clases tengan características de una clase padre. Esta clase se llamará hijo. La herencia la expresaremos con la palabra reservada extends.

Es muy conveniente utilizar require_once cuando queremos utilizar herencia e incluir clases que están en otros archivos.

Dentro de nuestra clase hijo podemos sobrescribir algún método del padre, esto es un concepto que conocemos como polimorfismo. Lo que polimorfismo quiere decir es que tendremos un método que va a funcionar de acuerdo con su contexto donde es llamado. 


                  require_once('clasePadre.php');

                  class claseHija extends clasePadre {
                    public function __construct($parametro1, $parametro2){
                      #Constructo del padre
                      parent::__construct($parametro1, $parametro2);
                    }

                    #Polimorfismo
                    public function nombreFuncionPolimorfismo(){
                      ...
                    }
                  }

Interfaces

Una Interface es un tipo abstracto que es usado para definir o especificar el comportamiento de ciertas clases. Los métodos de las interfaces no tienen implementación (Solo se describe el "enunciado"). Las interfaces no pueden ser instanciadas, sino implementadas por aquellas clases que tienen el comportamiento definido en la interfaz (es) que esta implementa.

Las interfaces son usadas para implementar las similitudes que varios tipos de clases comparten.

Con las interfaces se puede simular la herencia múltiple. 


                  #Definiendo la interfaz
                  interface PrintableInterface{
                    public function  getDescription();
                  }

                  #Usando la interfaz
                  class NombreClase implement PrintableInterface{
                    ...
                  }
                  
                  #Especificar el tipo de datos que esperamos en una función
                  function printElement(PrintableInterface $element){
                    ...
                  }

Namespaces

Esta es una forma de mantener únicos los nombres de los archivos en el mismo directorio.

Para declarar un espacio de nombres privado se utiliza la palabra reservada namespace.


                namespace App\Models;

                #Para usar una clase en un archivo diferente
                use App\Models\NombreClase;
                use App\Models\{NombreClase, OtraClase};

Composer

Composer es un manejador de dependencias de PHP, no solo nos ayudara a traer librerías de tercero al proyecto además va a implementar el estándar PSR4 (uno de los estándar para escribir código php) que nos va a permitir tener el cargado de archivos automático.

composer.phar será un documento que nos servirá para manejar las dependencias en PHP, son archivos de un empaquetado completo hecho en php, esta va muy de la mano con otro archivo llamado composer.json. 


                
                #Instalar composer
                #1. Pegar el archivo composer.phar en la carpteta del proyecto.
                #2. Crear composer.json
                
                {
                  #Configurando el autoload
                  "autoload": {
                    "psr-4" : {
                      "App\\": "app/"
                    }
                  },
                  "require": {}
                }
                
                #3. Ejecutar composer desde la consola
                php composer.phar install

                #4. Llamar el autoload en la clase
                require_once ('vendor/autoload.php');

ORM

Un ORM, que significa Object Relational Mapping, es un concepto en el cual vamos a crear dentro de nuestro código algunos modelos basados en las tablas de nuestra base de datos.

Una principal característica de un ORM es que hace más transparente las conexiones a PostgreSQL y MySQL, además nos protege de algunas vulnerabilidades de SQL y facilita algunas validaciones a la información.

PDO (PHP Data Object): Permite estandarizar como trabajar con diferentes sistemas manejadores de bases de datos.

ORM (Object Relational Mapping): Permite crear modelos/objetos basados en las tablas que tenemos en nuestra base de datos. Esto permite que la conexión sea más transparente con MySQL/PostgreSQL. Tiene ya validaciones básicas.

Formularios

Podemos enviar información desde un formulario a través de diferentes métodos, GET o POST. Para acceder a esta información desde PHP llamaremos a $_GET y $_POST, estas son variables super globales.

Eloquent

Eloquent es una libreria de laravel, funciona como un ORM, que se utiliza para hacer conexiones con la base de datos de manera más "fácil". 


                #Instalar eloquent
                composer require illuminate/database

                php composer.phar require illuminate/database

Configurar el acceso a la base de datos.


                use Illuminate\Database\Capsule\Manager as Capsule;
                
                $capsule = new Capsule;

                $capsule->addConnection([
                    'driver'    => 'mysql',
                    'host'      => 'localhost',
                    'database'  => 'nombreBaseDatos',
                    'username'  => 'root',
                    'password'  => '',
                    'charset'   => 'utf8',
                    'collation' => 'utf8_unicode_ci',
                    'prefix'    => '',
                ]);

setAsGlobal: Hacer todo como si estubiera en el contexto global 


                $capsule->setAsGlobal();

bootEloquent: Inicializar el ORM.


                $capsule->bootEloquent();

Hacer que nuestras clases (cuyas instancias son las que se guardan en las tablas de bd), hereden de Illuminate\Database\Eloquent\Model.

Illuminate coge las columnas de la tabla, y las convierte en atributos de la clase, es decir, de manera abstracta al extender la clase de Model. 


                use Illuminate\Database\Eloquent\Model
                class nombreClase extends Model{
                  ...

                  protected $table = 'nombreTablaBD';
                  
                  #Atributos que existen, pero de forma dinámica por el ORM,
                  estan mapeados en la tabla, pero no en la clase.  
                  #public $title;
                  #public $description;
                  #public $months;
                  
                }

Ahora para insertar datos a la base de datos, se crea una instancia de la clase y con el método global $_POST se toman los campos ingresados en el formulario, finalmente se usa el método save que viene heredado de Eloquent\Model. 


                $clase = new Clase();
                $clase->campo1 = $_POST['campo1'];
                $clase->campo2 = $_POST['campo2'];
                $clase->save();
