Curso práctico de Wordpress

Wordpress es un gestor de contenido CMS escrito en PHP, en el cual podemos gestionar contenido como texto o imagenes sin tener que saber código. Wordpress funciona con Mysql y como licencia maneja la GPL eso significa que es de código abierto.

Caracteristicas Principales

  • Autoadministrable: Permite administrar contenido y configuración sin usar código.
  • Gestión de usuarios: Permite asignar roles a los usuarios.
  • API REST: Permite generar operaciones mediante API, puede funcionar también como un Backend y podemos gestionar el Frontend desde otra tecnología.
  • Flexible: Permite personalizar tanto sus diseños como sus funcionalidades.
  • Actualizaciones: Está en constante crecimiento.
  • Comunidad: Cuenta con una gran comunidad que aporta a su desarrollo.

Temas, archivos principales

  • index.php: Archivo principal del tema y página de inicio.
  • style.css: Archivo principal de estilos.
  • front-page.php: Es la vista que carga por defecto como página de inicio.
  • function.php: Archivo central de todas las funcionalidades del tema. Nos permite con nuestro código ampliar las funcionalidades del código fuente de Wordpress.
  • page.php: Es la vista que carga por defecto para las páginas que se crean en Wordpress.

Hooks

Los hooks son funciones de Wordpress que nos permiten agregar código propio al código fuente de Wordpress, esto con el fin de evitar manipular directamente el código fuente y generar errores o perder las modificaciones por alguna actualización. Tipos de hooks:

Action: Nos permite ejecutar una función personalizada en un punto específico del código fuente de Wordpress. 


                  function holaMundo(){
                    echo "Hello world";
                  }
                  add_action('wp_head', 'holaMundo')
                  // add_action($hook, $function)

Filter Permite ejecutar una función personalizada en un determinado punto, pero la función que usemos tendrá un parámetro de entrada, dentro de nuestra función haremos modificaciones al parámetro y finalmente lo retornaremos modificado. 


                  function upperTitle($title){
                    return strtoupper($title);
                  }
                  add_filter('the_title', 'upperTitle')
                  // add_filter($hook, $function)

Roles de usuario

Se pueden presentar casos en los que los roles por defecto no alcancen o sean demasiado restrictivos, para eso WordPress brinda la posibilidad de crear, eliminar y modificar roles.

Para crear un rol de usuario podés utilizar la función add_role() que nos provee WordPress, la tenés que utilizar dentro de una función y asignarla al hook init. Esto tiene que hacerse desde el archivo functions.php. 


                    function add_administrador_tema_role() { //nombre de nuestra función, puede ser el nombre que quieras
                      add_role(
                          'administrador_tema', //Nombre de role.
                          'Administrador Tema', //Nombre que se visualará en la creación o página de opciones de usuarios.
                         array(    
                              'read' => true, //Permite el acceso al dashboard del adminitrador.
                              'switch_themes' => true, //Permite el cambio de temas.
                              'edit_themes'   => true, //Permite editar archivos desde el administrado de archivos del tema.
                              'edit_theme_options' => true, //Permite modificar Widgets,Menús, Personalizar.
                              'install_themes'    => true,  //Permite instalar temas nuevos.
                              'update_themes' => true, //Permite actualizar temas instalados.
                              'delete_themes' => true, //Permite eliminar temas.
                  
                              )   //Array con las capabilities
                      );
                    }
                  
                    //add_action(Hook, Nombre de la función)
                    add_action('init', 'add_administrador_tema_role');
